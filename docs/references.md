# References

These projects shaped the initial implementation direction.

## Browser Use

- URL: https://github.com/browser-use/browser-use
- Takeaway: Playwright-backed browser agents benefit from explicit browser state, replayable actions, and provider flexibility.

## Stagehand

- URL: https://github.com/browserbase/stagehand-python
- Takeaway: Robust automation should run deterministic actions first and use AI only when page changes or semantics require it.

## LaVague

- URL: https://github.com/lavague-ai/LaVague
- Takeaway: Natural-language web actions should produce constrained browser operations, not unconstrained code execution.

## HyperAgent

- URL: https://github.com/hyperbrowserai/HyperAgent
- Takeaway: A Playwright-first agent can expose higher-level natural-language actions while preserving browser automation primitives.
